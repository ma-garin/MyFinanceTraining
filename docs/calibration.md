# 較正（Calibration）エンジン — 方法論

「投資仮説OS」を世界水準にするための第1の柱。
**予測（確信度）と実際の結果を突き合わせ、「自分の予測がどれだけ当たるか」を定量化する。**

ニュース連想からの仮説は、確率を伴わなければ単なる物語にすぎない。
ここで実装する手法は、気象予報・スーパーフォーキャスティング（Tetlock）・
意思決定科学で標準的に使われるものをそのまま日本株仮説に適用したもの。

## 入力データ

各仮説に以下を付与する:

- **確信度 `confidence`（0–100%）**: 仮説が成立すると考える主観確率（予測）
- **各連想リンクの条件付き確率 `probability`（0–100%・任意）**: 前リンク成立下でこのリンクが成立する確率
- **結果確定 `resolution`**: hit（的中）/ miss（外れ）の最終結果

確信度と結果の両方が揃った仮説のみが較正の対象になる。

## 指標

### 1. Brier スコア
```
BS = (1/N) Σ (f_i − o_i)²        f: 予測確率(0–1), o: 結果(0/1)
```
0 = 完璧、0.25 = 無情報なコイン投げ（p=0.5）相当。**低いほど良い。**

### 2. Murphy 分解
```
BS ≈ 信頼度(reliability) − 分解能(resolution) + 不確実性(uncertainty)
```
- **信頼度**: 予測確率と実現頻度のズレ。**低い**ほど較正が良い。
- **分解能**: 各確率帯が基準率からどれだけ分離しているか。**高い**ほど選別眼がある。
- **不確実性**: `ō(1−ō)`。基準率に内在し制御不能。

→ 較正を直すなら「信頼度」を下げる（自信を抑える）。
　 選別眼を磨くなら「分解能」を上げる（当たり外れをはっきり予測する）。

### 3. Brier Skill Score (BSS)
```
BSS = 1 − BS / BS_ref      BS_ref = ō(1−ō)（常に基準率を予測した場合）
```
- BSS > 0: 基準率を上回る予測力（スキルあり）
- BSS = 0: 基準率と同等
- BSS < 0: 基準率を下回る（予測が逆効果）

### 4. 信頼度ダイアグラム（reliability diagram）
予測確率(x軸) vs 実現頻度(y軸)。点が対角線上にあれば較正良好。
対角線より下＝自信過剰、上＝自信不足。点の大きさ＝件数。

### 5. カテゴリ別ベイズ基準率（Beta-二項）
弱情報事前分布 `Beta(1, 1)` を各カテゴリの的中/外れで更新:
```
事後分布 = Beta(1 + 的中数, 1 + 外れ数)
事後平均 = (1 + 的中) / (2 + 的中 + 外れ)
```
少数標本でも暴れない基準率推定。95%信用区間は正規近似（個人用途で十分）。
これにより「地政学イベントは過去◯%しか当たっていない」といった
**カテゴリ別の地に足のついた事前確率**を蓄積できる。

### 6. 連想チェーンの積確率
各リンクの条件付き確率の積:
```
P(chain) = Π P(link_i)
```
これと主観確信度を比較し、**20pt以上の乖離**があれば警告する。
「自分のロジックの帰結（積確率）より確信度が高すぎる/低すぎる」を検出し、
自信過剰・過小を入力時点で自己点検できる。
